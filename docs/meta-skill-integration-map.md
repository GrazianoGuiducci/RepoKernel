# Meta-Skill Integration Map

Status: neutral transfer map

## Purpose

RepoKernel can learn from local meta-skills without copying private D-ND state.
This map separates transferable method from private residue.

## Transfer Rule

```text
local skill experience -> neutral function -> RepoKernel faculty or doc -> generated Project Kernel behavior
```

Do not copy local node memory, private paths, business strategy, client context,
runtime details, secrets or tool-specific assumptions into public RepoKernel
surfaces.

## New Candidate Selected

`clarity-cleanup-steward` is added because dirty or unclear files directly
reduce reentry quality and action clarity. The neutral method is safe:
classification and proposal first, cleanup only after an explicit owner gate.

## Boundary

The map is not a promotion packet by itself. A local skill becomes a RepoKernel
skill only when it has:

```text
neutral function;
private residue removed;
clear owner surface;
validation path;
registry entry;
receipt.
```
