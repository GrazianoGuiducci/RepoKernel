Reviewable project kernels without silent target writes.

What It Is

RepoKernel is a source-to-kernel compiler for AI-assisted project continuity. It is built around explicit contracts, evidence and review gates instead of background automation.

Contract First

It links a SourceManifest, ProjectModel and SeedSpec before a plan can be generated. The inputs must be authorized and validated.

Target Bound

Plans are tied to a target snapshot and can stage proposed files outside the target repository for human review.

No Silent Apply

The current line intentionally has no apply command, no runtime, no daemon and no credential handling.

The Flow

RepoKernel is meant to make the movement from intent to proposed project kernel inspectable at each step.

1 Authorized Sources

Operator-approved materials, not scraped private context.

2 ProjectModel

Mission, assertions, boundaries and unknowns.

3 SeedSpec

Reviewed specification with hashes and compatibility.

4 GenerationPlan

Deterministic proposed operations bound to inputs.

5 Stage

Files rendered into a separate review directory.

6 Review Gate

Human/operator decision before any future write-capable layer.

Try The Safe Path

The smoke path uses included minimal fixtures and writes only local output files plus an explicit staging directory. 

PYTHONPATH=src python -m repokernel.cli validate-spec --kind seed-spec --input examples/minimal/seed-spec.json
PYTHONPATH=src python -m repokernel.cli validate-bundle --source-manifest examples/minimal/source-manifest.json --project-model examples/minimal/project-model.json --seed-spec examples/minimal/seed-spec.json
PYTHONPATH=src python -m repokernel.cli plan --seed-spec examples/minimal/seed-spec.json --source-manifest examples/minimal/source-manifest.json --project-model examples/minimal/project-model.json > plan.json
PYTHONPATH=src python -m repokernel.cli stage --plan plan.json --output-dir ./repokernel-staging

Evidence And Limits

RepoKernel can be inspected publicly as a research/experimental repository. It should not be presented as an installer or autonomous modifier.

Current Evidence

  • Local unit suite: 47 passed on 2026-06-26.
  • Distribution verification: valid locally on 2026-06-26.
  • Hosted GitHub Actions CI: passing on main.
  • No-write case study replay: before_hash equals after_hash.

Blocked Claims

  • No production readiness.
  • No arbitrary-repository safety guarantee.
  • No apply, daemon, runtime or credential handling.
  • No PR, issue, network or external action automation.
