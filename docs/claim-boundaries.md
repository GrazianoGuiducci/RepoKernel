# Claim Boundaries

RepoKernel uses "context awareness" in an operational sense.

It does not claim that an AI model or a repository is conscious. It claims that a repository can expose enough context for future AI sessions to reenter responsibly.

## Safe Public Claims

- RepoKernel helps an AI agent reenter a repository without starting from zero.
- RepoKernel adds a context layer: state, sources, boundaries, decisions, next moves and memory deltas.
- RepoKernel supports long-running AI-assisted projects across tools and sessions.
- RepoKernel can be used for code, writing, research, product and knowledge work.

## Claims To Avoid

- The repository is conscious.
- The system is autonomous without supervision.
- The pattern removes AI error.
- The project improves itself without review.
- Any scientific, market or adoption claim that has not been verified.

## Better Framing

Use practical language first:

```text
Your repository should tell the next AI where it is, what matters, what changed and what to do next.
```

Use deeper language only after the practical problem is understood.
