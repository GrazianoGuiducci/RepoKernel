# Feedback Requested

RepoKernel improves through real use, not only internal tests.

If you install or try RepoKernel, we would appreciate a short note about what
was useful and what was confusing.

## Please Share

```text
project type:
mode used: new_repository | existing_repository_retrofit | observe-and-propose
readiness level used: L0 | L1 | L2 | L3 contract
what RepoKernel helped with:
what was unclear:
what was missing:
what felt unnecessary:
where the guide or installer got in the way:
what you expected but did not get:
whether the generated plan felt safe:
whether it helped the next AI/coder reenter the project:
```

## Do Not Share

```text
credentials
tokens
private logs
client material
private source documents
.env files
full repository dumps
anything you do not have permission to disclose
```

## Useful Feedback Examples

```text
"The source atlas helped the next coding session understand the repo."
"The planner marked an existing AGENTS.md as propose_update, which prevented overwrite."
"The guide did not explain which command to run first."
"The generated state file was too generic for a research project."
"I needed an example for an existing Python package."
```

## Why This Matters

RepoKernel is meant to become useful across different project types:

```text
software repositories
AI/RAG projects
editorial projects
business operations
research/lab work
creator portfolios
```

Real feedback helps decide which guides, examples, installer flow, CLI commands
and safety checks should be improved first.
